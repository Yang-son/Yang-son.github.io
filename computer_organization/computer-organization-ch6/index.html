6.1 无符号数和有符号数

一、无符号数

应用：存储器地址、I/O地址、指令代码、逻辑操作数等

寄存器的位数：反应无符号数的表示范围

通常称寄存器的位数为机器字长

例，机器字长16位，无符号数表示范围：0~65535

二、有符号数

应用：相对地址值、立即数等

1. 机器数与真值

机器数：符号“数字化”的数

真值：带“+”或“-”符号的数

最高位为符号位：0表示正，1表示负

使用一位或多位二进制数表示符号

区分定点数和浮点数：小数点是否固定

2. 原码表示法

  • 整数原码定义

    x为真值，n为整数的位数

  • 小数的原码表示

    x为真值

0在原码中有两种表示方法：全0或者10000000

即：在原码的有效编码中，“0”占两个编码值

3. 反码表示法

按书上的说法，反码通常用来作为原码与补码相互转化的中间过渡

  • 整数反码定义

    x为真值，n为整数的位数

  • 小数反码定义

    x为真值，n为小数的位数

    $[x]_{反}$有0,0000和1,1111两种形式

4. 补码表示法

  • 整数补码定义

    x为真值，n为整数的位数

  • 小数补码定义

    x为真值

  • 求补码的快捷方式：真值为正取自身，真值为负则取反再+1

小结

  1. 最高位为符号位，书写上用“,”（整数）或“.”（小数）将数值部分和符号位隔开

  2. 对于正数，原码 = 补码 = 反码

  3. 对于负数，符号位为1，其数值部分：

    原码：自身

    补码：原码除符号，位外每位取反末位加1

    反码：原码除符号位外每位取反

  4. 关于原码、反码、补码的表示范围

    整数：

    • 原码：$–(2^n–1) \leq x \leq 2^n–1$

    • 反码：$–(2^n–1) \leq x \leq 2^n–1$

    • 补码：$–2^n \leq x \leq 2^n–1$

      补码能比原码和反码多表示1个数，因为补码唯一表示了0（整数、小数皆如此）

    小数：

    • 原码：$–(1-2^{-n}) \leq x \leq 1-2^{-n}$
    • 反码：$–(1-2^{-n}) \leq x \leq 1-2^{-n}$
    • 补码：$–1 \leq x \leq 1-2^{-n}$

6.2 数的定点表示和浮点表示

定点数是重点，浮点数作了解

注意：

  1. 浮点数的尾数S不能为全0

一、定点表示

小数点固定在某一位置的数为定点数，有以下两种格式。

小数点位于数符和第一数值位之间时，机器内的数为纯小数；

小数点位于数值位之后时，机器内的数为纯整数。

在定点机中，由于小数点的位置固定不变，故当机器处理的数不是纯小数或者纯整数时，必须乘上一个比例因子，否则会产生“溢出”

关注定点机的表示范围，在前面也有所提到

规格化浮点：浮点数表示成规格化形式后，其精度最高。

1. 浮点数的表示形式

采用这种数据格式的机器称为浮点机。

浮点数由阶码j和尾数S两部分组成。

  • 阶码是整数，阶符和阶码的位数m合起来反映浮点数的表示范围及小数点的实际位置；

浮点数阶码小于最小阶码时，称为下溢。

发生下溢时，溢出的数绝对值很小，将尾数各位强置为0，按机器零处理，机器继续运行

6.3 定点运算

包括移位、加、减、乘、除

一、移位运算操作

移位对于计算机有很大的实用价值：可以采用移位和加法结合，实现乘（除）运算

需要考虑的问题：机器数的字长往往是固定的，当机器数左移n位或者右移n位时，必然会使其n位低位或n位高位出现空位。那么对空出的空位应该填补0还是1呢？

1. 算术移位方法

对有符号数的移位称为算术移位

算术左移、右移时符号位不变

  • 对于正数，由于原码、补码、反码都等于真值，故移位后的空位均以0添之

    目的：让填补代码后的代码与真值相等

  • 对于负数

    • 原码添0

      符号位不变，数值位与真值相同，以0添之

    • 反码添1

      负数的原码除符号位外，各位与负数原码相反，以1添之

    • 补码左移添0，右移添1

      从低位向高位找到第一个“1”时，其左边各位与对应的反码相同，右边各位与原码相同

      故补码左移时，因空位出现在低位，添补代码与原码相同，添0

      补码右移时，空位在高位，添补代码与反码相同，添1

例题

正数的算术移位运算

对于原码、补码、反码，无论左移还是右移，都添0

负数的算术移位运算

  • 原码：左移和右移都添0

  • 补码：左移添0，右移添1

  • 反码：左移和右移都添1

2. 算术移位的原理

符号位保持不变：符号位的输出接到输入端

  • 讨论移位丢掉1的情况

    算术移位的意义：左移乘2；右移除以2

    • 真值为正：

      • 左移丢1：最大的数丢失，结果出错

        如：0,1000001左移丢1，得到0,0000010。从65变成2（理论上乘2，变成130）

      • 右移丢1：最小的数丢失，结果影响精度

        如：0,1000001右移丢1，得到0,0100000。从65变成32（理论上除以2的结果为32.5——类似int的除法）

    • 负数的原码：数值部分同正数原码，因此左移和右移丢1的情况相同

    • 负数的补码：

      与补码算术移位填补代码的原理相同

      • 左移丢1：高位类似反码，丢1对应着原码丢0，无影响
      • 右移丢1：低位类似原码，丢1影响精度

    • 负数的反码：数值部分与原码相反，丢1对应原码丢0，无影响，结果正确

思考：移位的硬件如何配置

算术左移符号位SF移到进位/借位符号位CF中；右移最低位，移位到CF中。

3. 逻辑移位的原理

无符号数，所有位参加，添补0

三、乘法运算

有的机器使用硬件乘法器直接完成乘法运算；有的机器没有乘法器，用软件编程实现乘法运算的方法

1. 分析笔算乘法

包含被乘数A的多次左移，以及4个位积的相加运算

2. 笔算乘法改进

A和B按照权重进行打开

加一位，右移一位

3. 改进后的笔算乘法过程

两个寄存器：部分积和乘数。把符号位去掉，绝对值相乘。最后空出最高位，用于符号位。

每次执行的过程：①若乘数最右位为1，部分积加上被乘数，部分积右移（末位移到乘数寄存器的高位）；②乘数右移，继续执行①，直到原乘数全部移出

小结

全加器的个数与位数有关

运算过程：

①乘法运算可用移位和加法来实现

两个4位数相乘，总共需要进行4次加法运算和4次移位

②由乘数的末位确定被乘数是否与原部分积相加，然后右移一位，形成新的部分积；同时，乘数也右移一位，由次低位作新的末位，空出最高位放部分积的最低位

③每次做加法运算时，被乘数仅仅与原部分积的高位相加，其低位被移至乘数所空出的高位位置

6.4 浮点四则运算

了解内容，需知道运算过程

掌握程度：浮点数运算的步骤。

加减：把阶放成相同

乘除：两个定点数运算

一、浮点加减运算

1. 对阶

对阶的目的是使两操作数的小数点位置对齐，即使两数的阶码相等

为此，首先求出阶差，再按小阶向大阶看齐的原则使阶小的尾数向右移位，每右移1位，阶码加1，直到两数的阶码相等为止

右移的次数正好等于阶差。

尾数右移时可能会发生数码丢失，影响精度

2. 尾数求和

将对阶后的两个尾数按定点加（减）运算规则进行运算。

3. 规格化

当尾数最高数值位与符号位不同时，即为规格化形式。

但对S < 0时，有两种情况需特殊处理

左规和右规

  • 尾数出现00.0xx…x或11.1xx…x时，需左规

    左规时，尾数左移1位，阶码减1

  • 尾数出现01.xx…x或10.xx…x时，表示尾数溢出，可通过右规处理

    右规时，尾数右移1位，阶码加1

例 6.27

4. 舍入

在对阶和右规的过程中，可能会将尾数的低位丢失，引起误差，影响精度。

为此可用舍入法来提高尾数精度

常用的舍入方法：

  • “0舍1入”法

    类似于十进制运算的四舍五入

    尾数右移时，被移去的最高数值位为0，则舍去；

    被移去的最高数值位为1，则在尾数末位加1

  • “恒置1”法

    尾数右移时，无论丢掉的最高数值位是1还是0，都使右移后的尾数末位恒置1

    这种方法同样有使尾数变大和变小的两种可能

5. 溢出判断

在浮点规格化中已经指出，当尾数之和（差）出现01.xx…x或10.xx…x时，并不表示溢出，只有将此数右规后，再根据阶码来判断浮点运算结果是否溢出

6.5 算术逻辑单元

解决问题：移位、算术运算

一、ALU电路

回顾ch1：运算器最少包括3个寄存器和一个算术逻辑单元（ALU）。其中ACC为累加器，MQ为乘商寄存器，X为操作数寄存器

工作原理：算术逻辑单元的核心部件是加法器和寄存器。当需要完成逻辑运算时，势必需要配置相应的逻辑电路，而ALU电路是既能完成算术运算又能完成逻辑运算的部件。

  • $A_i$和$B_i$为输入变量；
  • $k_i$（74181中为$S_i$）为控制信号，$k_i$的不同取值可决定该电路做哪一种算术运算或哪一种逻辑运算；
  • $F_i$是输出函数

ALU参考资料：算术逻辑单元（ALU的功能和基本结构、一位全加器，串行加法器，并行加法器）

74181：能完成4位二进制代码的算逻运算部件

具有两种工作方式——正逻辑和负逻辑

$S_3-S_0$控制译码

74181芯片的核心部件是一个容量为16字的双端口RAM和一个高速ALU电路。

RAM可视为一个由16个寄存器组成的寄存器堆——时序逻辑电路，有锁存器

二、快速进位链

随着操作数位数的增加，电路中进位的速度对运算时间的影响也越来越大。

为了提高运算速度，通过对进位过程分析设计快速进位链

1.并行加法器

并行加法器由若干个全加器组成

n+1个全加器级联组成了一个n+1位的并行加法器

2.串行进位链

串行进位链是指并行加法器中的进位信号采用串行传递

注意串行进位链的延时

n位全加器产生进位的全部时间为$2nt_y$

3.并行进位链（先行进位，跳跃进位）

并行进位链是指并行加法器中的进位信号是同时产生的，又称先行进位、跳跃进位等。

理想的并行进位链是n位全加器的n位进位同时产生，但实际做不到，实现有困难。

二级进位，最慢的是与或非门加上与非门，一共2.5$t_y$

电路复杂，速度快
