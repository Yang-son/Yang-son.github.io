计算机组成原理：第7章 指令系统

发表于 2022-05-05 更新于 2022-05-27 分类于 2022 Spring ， 计算机组成原理 阅读次数：
本文字数： 5.4k 阅读时长 ≈ 18 分钟

计算机的最高指挥：晶振——产生指令。

计算机在指令驱动下运行。指令组成程序

7.1 机器指令

机器语言：由一条条语句构成，每一条语句又能准确表达某种语义

机器指令：每一条机器语言的语句

指令系统：全部机器指令的集合

一、指令的一般格式

1.操作码

反映机器做什么操作

用来指明该指令所要完成的操作，

如：加法、减法、传送、移位、转移等

位数反映操作种类（机器允许的指令条数）

如，操作码占7位，最多包含$2^7=128$条指令

  • 三地址指令：操作码每减少一种，可多构成$2^4$种二地址指令
  • 二地址指令：操作码每减少一种，可多构成$2^4$种一地址指令

2.地址码

用来指出该指令的源操作数的地址（一个或两个）、结果的地址以及下一条指令的地址。

“地址”可以是主存的地址，也可以是寄存器的地址，I/O设备的地址

1. 四地址指令

（注意书上和PPT中访存次数有误）

5次访存：取$A_1$，取$A_2$，写$A_3$，取下一条指令的地址$A_4$，还需要从指令地址中取指令

4个地址字段各占6位，指令操作数直接寻址范围为$2^6=64$（按老师的说法，应写成：$0-2^6-1$）

2. 三地址指令

现代计算机使用PC代替$A_4$，形成三地址

4次访存：取$A_1$，取$A_2$，写$A_3$，取指令（程序计数器PC）

3个地址字段各占8位，指令操作数直接寻址范围：256（0-255）

比较四地址和三地址：区别在于下一条指令地址。前者把下一条地址放在$A_4$字段，需要访存；后者使用程序计数器PC存放，CPU内部硬件，不访存。

3. 二地址指令

4次访存：取$A_1$，取$A_2$，写$A_1$（或$A_2$），取指令

3次访存：结果暂存在累加器ACC中，不用写$A_1$或$A_2$

2个地址字段各占12位，直接寻址范围：4096（0 - 4095）

4. 一地址

使用ACC代替$A_1$或$A_2$

2次访存：取$A_1$，取指令 —— ACC在CPU的运算器中，不访存

地址字段16位，直接寻址范围：16M（0 - 16M-1）

5. 零地址

空操作（NOP）、停机（HLT）——只有操作码

子程序返回（RET）、中断返回（IRET）——没有地址码

没有地址码，操作数的地址隐含在堆栈指针SP中

总结：

  • 使用硬件资源（如PC、ACC）承担指令字中需指明的地址码，可在不改变指令字长的前提下，扩大指令操作数的直接寻址范围；

二、指令字长

指令字长取决于$\begin{cases} 操作码的长度\\ 操作数地址的长度 \\ 操作数地址的个数 \\ \end{cases}$

  1. 指令字长固定

早期的计算机，访问某个存储单元便可取出一条完整的指令或一个完整的数据。控制方式简单

  1. 指令字长可变

    按字节的倍数变化

    为了提高指令的运行速度和节省存储空间，通常尽可能把常用的指令（如：数据传送指令、算逻运算指令等）设计成单字长或短字长格式的指令

小结

7.2 操作数类型和操作类型

一、操作数类型

常见的操作数类型：地址、数字、字符、逻辑数据

  • 地址：无符号的整数。涉及操作数地址的计算

  • 数字：定点数、浮点数和十进制数

  • 字符：普遍采用ASCII码

  • 逻辑数据：用于逻辑运算，0和1的数字组合

    注意与算术数字的区别

    例，ASCII码中的0110101（54，表示十进制5），与逻辑数0001111做与运算得到NB-CD短十进制码0101

二、数据在存储器中的存放方式

计算机数据存放在存储器或寄存器中，寄存器位数反映机器字长

一般取字节的整数倍（1、2、4、8、……）

”边界对齐“是为了便于硬件实现

三、操作类型

1. 数据传送

包括：寄存器与寄存器、寄存器与存储单元、存储单元与存储单元之间的传送。

如：从源到目的之间的传送、对存储器读（LOAD）和写（STORE）、交换源和目的的内容、置1、清0、进栈、出栈等

2. 算术逻辑操作

3. 移位操作

移位操作所需时间远比乘除操作执行时间短，用来代替简单的乘法和除法运算

  • 算术移位：对有符号数的乘/除以2的运算
  • 逻辑移位：无符号数
  • 循环移位（带进位和不带进位）

4. 转移

多数情况下，计算机按照顺序执行程序的每条指令，但有时需要改变顺序，采用转移类指令来完成

分为：无条件转移、条件转移、跳转、过程调用与返回、陷阱等

  • 无条件转移

    不受任何条件约束，可直接把程序转移到下一条需执行指令的地址

  • 条件转移

    根据当前指令的执行结果来决定是否需要转移。

    若条件满足，则转移；不满足，则继续按顺序执行。

    一般机器都能提供一些条件码，这些条件码是某些操作的结果

    如：对于

    零标志位ZF，结果为0，ZF=1；

    负标志位SF，结果为负，SF=1；

    溢出标志位OF，结果有溢出，OF=1；

    进位标志位CF，最高位有进位，CF=1；

    奇偶标志位PF，结果呈偶数，PF=1；

  • 调用和返回

    CALL和RETURN配合使用

    CALL用于转到子程序入口，RETURN用于子程序执行完返回原程序断点

    类似中断

    在编写程序时，有些具有特定功能的程序段会被反复使用。为避免重复编写，可将这些程序段设定为独立子程序，当需要执行某子程序时，只需子程序调用指令即可。

  • 陷阱（Trap）与陷阱指令

    陷阱是一种意外事故的中断

    如，机器在运行中，可能会出现电源电压不稳定、存储器校验出差错、输入输出设备出现了故障、用户使用未被定义的指令、除数出现0、运算结果溢出、特权指令等

    一旦出现意外故障，计算机就发出陷阱信号，暂停当前程序的执行，转入故障处理程序，进行相应的故障处理。

5.输入输出

针对I/O单独编址的计算机。完成外设寄存器与CPU寄存器之间的数据读入或输出

AL是8bit，AX是16bit

输入：端口送入CPU

6.其他

包括等待指令、停机指令（STP）、空操作指令、开中断指令、关中断指令、置条件码指令等

思考：已知指令的顺序寻址、中断隐指令、开机第一条指令

我的理解：开机第一条指令位于0xFFFF0，是一条跳转类指令，内容为BIOS入口地址（我认为是计算机第一条指令）。CPU之后进行硬件自检和创建中断向量表。之后CPU收到int 0x19中断（自检之后的第一条指令），启动BIOS。

由此，在计算机执行完第一条指令后，中断向量表尚未建立，中断类型尚未标明，无法响应中断。在向量表建立后，EINT=1且I/O发出中断请求且MASK=0时，CPU才能在指令执行结束阶段响应中断。

开机第一条指令：自动去取CS:IP地址的指令。也就是位于FFFF:0000（FFFF0）地址的指令

对于8086CPU来说，进行硬件初始化，会把所有的CPU置为0，除了代码段寄存器CS会置为FFFF

复位和加电的功能等效

0xFFFF0 指令：跳转到F000:EO5B

参考资料：开机后的第一条指令、8086-1-计算机的启动过程

中断隐指令怎么找

  • 中断处理的本质类似于调用，响应中断$\to$运行中断处理程序$\to$恢复被中断程序，整个流程不会导致被中断过程的连贯性。也可以主动调用中断处理程序，int 0xN，只不过就是你不知道中断处理程序的入口地址在哪，入口保存在中断向量表(intel 16位机)、中断描述符表IDT(intel 32位机)。

  • CPU收到中断信息后，首先引发中断过程，由硬件完成，没有指令对应。即中断隐指令。以8086为例，中断过程做的事：取中断类型码N；EFLAGS进栈；TF=0，IF=0；CS进栈；IP进栈；设置CS、IP为中断处理程序的入口。

  • 而中断处理程序要做的第一件事就是保存通用寄存器的内容。

参考资料：操作系统中断处理时的程序断点保存在哪里了？

一、指令寻址

分为顺序寻址和跳跃寻址两种

  • 顺序寻址：(PC) + 1$\to$ PC
  • 跳跃寻址：JMP

二、数据寻址

指令字中必须设一字段来指明属于哪一种寻址方式——寻址特征位，记作M

指令的地址码字段通常都不代表操作数的真实地址，把它称为形式地址，记作A

操作数的真实地址称为有效地址，记作EA

假设指令字长、存储字长、机器字长均相同

1. 立即寻址

特点：操作数本身设在指令字内，即：形式地址A不是操作数的地址，而是操作数本身，又称之为立即数

立即数是有符号数，以补码形式保存

优点：只要取出指令，便可立即获得操作数，在执行阶段不必再访问存储器

缺点：A的位数限制立即数的范围

2. 直接寻址

特点：指令字中的形式地址A就是操作数的真实地址EA，即：EA=A

有效地址由形式地址直接给出

优点：寻找操作数简单，不需要专门计算操作数的地址，在指令执行阶段对主存只访问一次。

缺点：A的位数限制了操作数的寻址范围，而且必须修改A的值才能修改操作数的地址。

3. 隐含寻址

操作数地址隐含在操作码或某个寄存器中

另一个操作数隐含在ACC中

4. 间接寻址

有效地址由形式地址间接给出。

类似于指针，有效地址是形式地址单元的数值

一次间址对应两次访存过程：

  • 第一次：取有效地址EA（操作数地址）
  • 第二次：获得操作数

设指令字长和存储字长均为16位，形式地址A为8位。则直接寻址$2^8$，间接寻址范围$2^{16}$

优点：

  • 扩大寻址范围

    因为A的位数通常小于指令字长，而存储字长可与指令字长相等

  • 便于编制程序

    方便完成子程序返回

缺点：指令执行期间多次访存，指令执行时间延长

5. 寄存器寻址

有效地址即为寄存器编号

在指令执行阶段无须访存，减少执行时间

地址字段只需指明寄存器编号，指令字较短，节省存储空间

6. 寄存器间接寻址

有效地址在寄存器内（访问寄存器内容），操作数在存储器

类似于存储器间接寻址

7. 基址寻址

设有基址寄存器BR。操作数有效地址等于形式地址+基址寄存器的内容

基址寄存器的内容通常由OS或者管理程序确定，在程序执行过程中其值不可变，而指令字中形式地址A可变。

主要用于为程序或数据分配存储空间。

  1. 采用专用寄存器作基址寄存器

  2. 采用通用寄存器$R_0$作基址寄存器

8. 变址寻址

与基址寻址类似

有效地址等于形式地址A与变址寄存器IX的内容相加之和

变址寄存器内容由用户设定，在程序执行过程中其值可变，但指令字的形式地址A不可变。

主要用于处理数组问题，设定A为数组首地址。

9. 相对寻址

有效地址是将PC的内容（当前指令的地址）与形式地址A相加而成

形式地址A是相对与当前指令的位移量，可正可负，用补码表示

常用于转移类指令，转移后的目标地址与当前指令有一段距离。

例，若位移量为8位，指令寻址范围：(PC) + 127 ~ (PC) - 128

可与间接寻址配合使用

10. 堆栈寻址

要求计算机中设有堆栈。堆栈既可用寄存器组（硬堆栈）来实现，也可利用主存的一部分空间作为堆栈（软堆栈）

堆栈指令调用内存（堆栈指令或者内存单元操作）

可见堆栈寻址也可视为一种隐含寻址，操作数地址隐含在SP中

就其本质可视为寄存器间接寻址——SP可视为寄存器，存放操作数的有效地址EA

7.5 RISC技术（重要）

RISC(Reduced Instruction Set Computer)，全称：精简指令系统计算机

CISC(Complex Instruction Set Computer)，全称：复杂指令系统计算机

RISC：硬件简单，寻址方式、指令砍掉，无法运行CISC程序，不兼容

RISC指令系统选取使用频度较高的一些简单指令，复杂指令的功能由简单指令的组合来实现。其指令长度固定，指令格式种类少，寻址方式种类少，只有取数/存数指令访问存储器，其余指令的操作都在寄存器之间进行，且采用流水线技术，大部分指令在1个时钟周期时间内完成。

控制器采用组合逻辑控制，不采用微程序控制

四、RISC和CISC 的比较

比较内容 RISC CISC
指令系统 简单、精简 复杂、庞大
指令数目 一般小于100 一般大于200
指令格式 一般小于4 一般大于4
寻址方式 一般小于4 一般大于4
指令字长 定长 不固定
可访存指令 只有取数(LOAD)/存数(STORE)指令
其余在寄存器内完成		 不加限制
各种指令使用频率 相差不大 相差很大
各种指令执行时间 绝大多数在一个周期内完成 相差很大
优化编译实现 较容易 很难
程序源代码长度 较长 较短
控制器实现方式 绝大多数为组合逻辑控制（硬布线控制） 绝大多数为微程序控制
软件系统开发时间 较长 较短
